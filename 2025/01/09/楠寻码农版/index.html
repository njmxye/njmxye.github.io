爬虫？

1. 爬虫概述

1.1 定义

用代码代替人去模拟浏览器或手机执行操作，如自动登录、数据采集等。

1.2 案例

  • 自动登录钉钉并定时打卡。
  • 从91网站下载图片/视频。
  • 在京东抢购茅台。

1.3 分析与模拟流程

  • 基于谷歌浏览器分析请求。
  • 用requests模块发送请求。

1.4 requests模块

  • 安装：pip3.11 install requests。
  • URL参数：发送GET/POST请求时，网址可能携带参数，可直接在URL中写或用params参数传递。
  • 请求体格式：
    • form表单格式：特征为谷歌浏览器抓包Form Data，请求头Content-Type为application/x-www-form-urlencoded; charset=UTF-8，可用data参数（字符串或字典形式）发送。
    • json格式：特征为谷歌浏览器抓包Request Payload，请求头Content-Type为application/json;charset=utf-8，可用data=json.dumps()或json参数发送。
  • Cookie：
    • 本质是浏览器存储的键值对，用于保存用户凭证。
    • 读取返回的Cookie：res.cookies.get_dict()。
    • 发送请求时携带cookie：可通过headers中的Cookie字段或cookies参数传递。
  • 响应体格式：
    • 原始字节：用于图片、文件、视频下载，通过res.content获取。
    • 普通文本：通过res.text获取。
    • 转换格式：若响应为JSON格式，可用res.json()自动转换为Python字典或列表，也可手动用json.loads(res.text)转换。

2. 数据提取

2.1 HTML格式与数据提取方法

  • HTML格式由标签组成，可根据标签名称或属性寻找标签。
  • 基于bs4模块提取数据：
    • 安装：pip3 install beautifulsoup4。
    • 使用：
      • 根据标签名称获取标签（find），获取多个标签（find_all）。
      • 根据属性获取标签。
      • 嵌套读取，先找到父标签，再在父标签中找子标签。

2.2 案例

  • X车网：获取所有汽车品牌列表。
  • 网易云音乐：采集标签并下载封面图片。

3. 验证码识别

3.1 图片验证码识别方法

基于Python的ddddocr模块识别图片验证码。

  • 安装：pip3.11 install ddddocr==1.4.9 -i https://mirrors.aliyun.com/pypi/simple/和pip3.11 install Pillow==9.5.0。
  • 识别方式：
    • 本地识别：读取本地图片文件，用ocr.classification(body)识别。
    • 在线识别：先请求获取图片，再用ocr.classification(res.content)识别。
    • 处理base64编码图片：先解码，再识别。

3.2 案例

  • X文街：实现登录认证，包括获取验证码地址、识别验证码、登录认证等步骤。

4. 自动化与模拟浏览器

4.1 Selenium自动化操作

  • 操作浏览器页面，实现点击、输入、滑动等操作。
  • 与逆向对比：
    • Selenium优点是简单，无需逆向，缺点是性能差，不利于批量实现。
    • 逆向优点是算法逆向后性能好且利于批量实现，缺点是语法难，js加密算法不易逆向。

4.2 Selenium基本操作

  • 模块与驱动：
    • 安装模块：pip3.11 install selenium。
    • 下载驱动：根据浏览器类型下载对应驱动，如谷歌驱动（不同版本有不同下载地址），并配置环境变量。
    • 快速使用示例。
  • 寻找标签：
    • 方法：find_element（查找单个标签）和find_elements（查找多个标签）。
    • 定位方式：By.ID、By.CLASS_NAME、By.TAG_NAME、By.XPATH等。
  • 执行操作：常见操作如点击（click）、输入（send_keys）。
  • 执行JavaScript：可用execute_script方法在页面执行js代码，如点击、获取cookie等操作。
  • 等待：
    • 隐式等待：设置全局等待时间，driver.implicitly_wait(30)。
    • 显式等待：等待某个元素加载成功后再执行操作，如使用WebDriverWait结合lambda表达式或自定义函数。
  • 获取值：找到标签后，可获取标签的文本（text）、属性（get_attribute）、值（value）等。
  • 源码+bs4：可结合bs4模块，通过driver.page_source获取页面源码后用bs4解析。
  • 携带Cookie：driver.add_cookie({'name': 'foo', 'value': 'bar'})。
  • IP检测和代理：若网站限制IP访问，可购买IP并在请求时添加代理，如通过opt.add_argument(f'--proxy-server={proxy_string}')配置。
  • 特征检测：有些网站检测浏览器特征，可通过配置ChromeOptions隐藏特征，如opt.add_argument('--disable-infobars')等。
  • 无头和其他配置：可设置无头模式（opt.add_argument('--headless')）及其他配置，如禁止策略化、解决报错、设置分辨率、禁用javascript等。
  • 截屏：找到标签后，可通过截图方法保存图片，如tag.screenshot("demo.png")。

4.3 案例

  • X东搜索：实现搜索功能，包括打开京东、输入关键词、点击搜索、查询列表等步骤，需注意添加代理和隐藏特征。
  • X麦网：实现搜索功能，注意访问频繁时可能提示验证码，后续需解决滑动验证、文字识别点选验证等问题。

5. 验证码破解

5.1 滑块验证码破解步骤

  • 获取验证码图片：
    • 打开网页，点击滑动拼图验证，开始验证后获取背景图片和缺口图片的URL。
    • 示例代码中通过WebDriverWait等待元素出现，然后用re.findall从元素的style属性中提取图片URL。
  • 识别图片：计算缺口左边的横坐标（滑动距离），有以下几种方法：
    • ddddocr：用slide_match方法识别，返回目标位置坐标。
    • opencv：通过边缘检测、模板匹配等方法计算距离。
    • 打码平台：将图片上传到打码平台，根据平台返回的结果获取距离。
  • Selenium滑动：用ActionChains模拟鼠标操作，先点击并按住滑块，然后根据计算的距离滑动，最后释放滑块。

5.2 案例

  • 极验滑块：详细步骤包括打开首页、点击滑动拼图验证、获取图片、识别图片、滑动滑块等，最终实现滑块验证。

5.3 中文识别点选验证码破解步骤

  • 获取图片：打开网页，点击文字点选验证，开始验证后获取要识别的目标图片和背景图片。
  • 目标识别：截图每个字符，用ddddocr识别，获取要识别的文字列表。
  • 背景坐标识别：
    • ddddocr：能识别背景文字，但默认识别率低，可搭建Pytorch环境训练模型提高识别率，用detection方法获取文字坐标。
    • 打码平台：将背景图片上传到打码平台，获取文字坐标。
  • 坐标点击：根据识别的文字和坐标，用ActionChains移动到相应坐标并点击。

5.4 案例

  • 极验中文识别点选：详细步骤包括打开首页、点击文字点选验证、获取图片、识别目标文字、获取背景文字坐标、点击验证码等，最终实现中文识别点选验证。

6. 网站登录与逆向分析

6.1 网站登录流程与逆向分析方法

  • 常见登录流程：
    • 方式1：第一次访问获取内容和Cookie（此时凭证无登录权限），第二次输入用户名和密码提交，携带第一次的Cookie，后台校验成功后赋予凭证登录权限，后续访问携带Cookie获取用户信息。
    • 方式2：第一次访问返回页面，第二次输入用户名和密码提交，后台校验成功后在响应体或Cookie中返回用户登录凭证，后续访问携带凭证获取用户信息。
  • 逆向分析：根据网站具体情况，分析登录请求，找到加密算法、参数生成方式等，然后用代码实现相同逻辑。

6.2 案例

  • XX军事网：分析请求属于第二种登录流程，实现自动登录包括发送登录请求、获取Cookie、访问个人中心并获取用户信息等步骤。
  • 北大未名BBS：
    • 分析：输入密码后密码被加密提交
    • 提交登录请求，获取响应和Cookie。

6.3 XX儿登录逆向分析

  • 逆向滑块请求：
    • 请求分析：滑块请求中data参数包含加密数据，由滑动轨迹序列化后AES加密生成，同时涉及浏览器环境相关操作。
    • 轨迹：根据分析的算法构建滑动轨迹信息，包括横坐标、纵坐标、时间等参数的计算。
    • AES加密：用Crypto.Cipher模块的AES类进行加密，需注意加密的参数设置，如密钥、模式、填充方式等。
    • 示例代码：整合轨迹生成和加密过程，发送滑块请求并获取响应。
  • 逆向提交sendLoginCode：
    • 请求分析：提交手机号信息时，bella参数加密算法未知，通过分析发现其由window.Bella函数生成。
    • 分析window.Bella：通过猜想、HTML验证、浏览器补环境实现等步骤，最终找到或还原window.Bella的算法，实现提交登录码请求。
  • 短信登录：输入手机号、获取验证码、输入验证码后，携带相关参数发送请求实现登录，同时获取登录后的Cookie等信息。

6.4 XX儿升级版登录逆向分析

  • 文本替换（混淆）：分析js代码中混淆的字符串，通过编写代码替换函数将混淆字符串还原为真实值，可选择单独替换或整体文件替换。
  • 算法分析：
    • 入口判断：找到window['Bella']的赋值函数，分析其入口判断逻辑，发现异常时返回固定值，可先尝试用固定值测试。
    • 测试成功：发现直接使用固定字符串可实现部分功能。
    • 深入分析：若固定值不行，则深入分析算法，包括_0x4dd553（拆分参数）、_0x15c356（获取页面相关信息）、_0x34877a（生成随机字符串）、_0xc21476（签名计算）、_0x34c54c（复杂函数处理）等部分的详细逻辑。
  • 算法还原：根据分析结果，用Python代码实现相同算法，包括hmac_sha1（签名计算）、_0x26cf34（复杂函数逻辑）、get_bella（整合生成bella参数）等函数，最终实现滑块和登录的整体测试。

7. TLS指纹校验与绕过

7.1 TLS指纹校验原理

  • 如今网站通信多基于Https协议，TLS握手阶段客户端发送的Client Hello数据包中的JA3指纹信息用于标识客户端。JA3指纹基于电脑的TLS版本、内置算法等计算，且不会随请求头修改或代理而变化。网站后台通过读取JA3指纹来判断客户端是否合法，若指纹异常则禁止访问。

7.2 查看指纹方法

  • 快速查看：访问https://tls.browserleaks.com/json获取TSL指纹信息。
  • Wireshark抓包：下载安装Wireshark，选择网卡，筛选目标IP，用浏览器或requests发送请求后，在抓到的数据包中找到Client Hello数据包查看指纹信息。

7.3 JA3指纹组成

  • JA3是对JA3 Fullstring进行md5加密的结果。
  • JA3 Fullstring由五段组成：
    • SSLVersion：表示SSL版本，如TLS 1.2（0x0303，转换为十进制为771）。
    • Cipher：表示TLS客户端支持的加密套件，将算法套件的十六进制表示转换为十进制后拼接。
    • SSLExtension：表示支持的TLS扩展，对应SSL中扩展的数字。
    • EllipticCurve：表示支持的椭圆曲线（ECC加密算法），将supported_groups对应的十六进制转换为十进制。
    • EllipticCurvePointFormat：表示支持的椭圆曲线格式，将ec_point_formats转换为十进制。

7.4 突破指纹方法及案例

  • 【PY】requests：
    • 原理：requests发送请求依赖urllib3，通过修改ciphers实现生成非默认ja3指纹。
    • 方式1：直接修改urllib3.util.ssl_.DEFAULT_CIPHERS。
    • 方式2：自定义HTTPAdapter，在init_poolmanager和proxy_manager_for方法中设置ssl_context。
    • 案例：可过部分网站（如案例3）的TLS校验机制。
  • 【PY】curl_cffi：
    • 原理：基于curl - impersonate开发，可完美模拟主流浏览器，通过impersonate参数指定模拟的浏览器类型。
    • 安装：pip install curl - cffi。
    • 案例：可成功访问案例1、2、3中的网站。
  • 【Go】CycleTLS：
    • 原理：基于Golang开发，可更靠近底层定制ja3指纹，直接自定义ja3 fullstring。
    • 安装：go get github.com/Danny - Dasilva/CycleTLS/cycletls。
    • 案例：可成功访问案例1、2、3中的网站，需按照指定格式设置Options参数。

8. Mitmproxy使用

8.1 Mitmproxy安装与配置

  • 安装报错及解决：在Python3.9.10虚拟环境中安装mitmproxy可能报错，需下载Visual C++ Build Tools for Visual Studio 2015并安装，然后重新执行pip install mitmproxy。
  • Https请求支持：
    • 启动Mitmproxy：执行mitmdump -q -p 8888 -s v1.py启动，v1.py中可定义请求和响应的处理逻辑。
    • 获取证书：可从本地C:\Users\Administrator\.mitmproxy获取或通过代理下载（访问http://mitm.it/）。
    • 安装证书：将获取的证书安装到电脑。

8.2 Mitmproxy请求与响应处理

  • 请求：

    • 读取请求：在request函数中可获取请求的各种信息，如url、host、path、query、cookies、headers、method、content等。
    • 修改请求：修改flow.request的相关属性，如url、headers、content等。
    • 拦截请求：可根据条件拦截请求并返回指定内容或直接kill当前请求。
    • 案例：免账号登录，通过修改请求的Cookie实现。

  • 响应：

    • 读取响应：在response函数中可获取响应的状态码、cookies、headers、content等信息。
    • 修改响应：修改flow.response的相关属性，如status_code、content、headers等。

9. 数美点选验证逆向分析

9.1 必备知识点

  • 动态无法调试解决方案：
    • 安装抓包工具Charles，用其代理网页请求。
    • 在Charles中配置，将返回的js地址替换为固定值，便于断点调试。
  • 文本替换：
    • 单独替换：通过subprocess.check_output执行node脚本，将混淆字符串转换为真实值。
    • 整体替换：编写代码读取js文件，找到混淆字符串的函数调用，替换为真实值后写入新文件。

9.2 整体请求分析

  • 配置：发送请求获取配置信息，如organization（固定）、lang、sdkver、captchaUuid（动态，需逆向）等参数。
  • 获取验证码：点击登录或验证失败后，请求获取验证码图片及相关信息。
  • 提交验证：点击验证码后自动提交验证请求，包含多个加密参数。

9.3 逆向过程

  • 配置：分析captchaUuid的生成算法（时间 + 随机字符串），用Python实现生成并发送配置请求。
  • 获取验证码：直接发送请求获取验证码信息，包括图片URL、文字顺序等。
  • 提交指纹：
    • 定位：分析触发提交请求的操作（如点击第4标记）。
    • 特殊替换：对js文件中的混淆字符串进行替换，便于分析。
    • 第4次请求：分析请求参数的生成逻辑，涉及多个函数调用和加密操作。
    • getEncryptContent：该函数用于对参数进行加密，包括DES加密和base64编码。

17. x龙逆向分析

17.1 动态调试难题

在进行网页逆向分析时，经常会遇到JavaScript文件动态加载的问题，导致无法在浏览器中直接断点调试。解决方案是使用代理工具如Charles，将返回的js地址替换为固定的值，从而使得调试成为可能。

Charles安装与配置

  • Charles安装：下载并安装Charles，注册并激活。
  • Charles证书：安装Charles证书到电脑，以便Charles能够抓取浏览器的请求包。
  • 页面替换：在Charles中配置，使得浏览器访问特定网址时返回指定的HTML数据，从而固定js文件的URL。

文本替换

  • 单独替换：使用Python的subprocess模块执行Node.js脚本来替换混淆的字符串。
  • 整体替换：编写Python脚本，读取js文件，识别并替换混淆字符串，输出新的js文件。

17.2 整体请求分析

  • 配置请求：分析获取验证码前的配置请求，包括organization、lang、sdkver、captchaUuid等参数。
  • 获取验证码：分析触发验证码显示的请求，获取验证码图片的URL。
  • 提交验证：分析提交验证码的请求，包括加密参数的生成和提交。

17.3 逆向过程

  • 配置逆向：分析captchaUuid的生成算法，通常为时间戳加随机字符串，用Python实现并发送配置请求。
  • 获取验证码逆向：直接请求获取验证码信息，包括图片URL和文字顺序。
  • 提交指纹逆向：
    • 定位：分析触发提交请求的操作，如点击验证码。
    • 特殊替换：对js文件中的混淆字符串进行替换，便于分析。
    • 第四次请求：分析请求参数的生成逻辑，涉及多个函数调用和加密操作。
    • getEncryptContent：该函数用于对参数进行加密，包括DES加密和base64编码。

17.4 x龙点选验证

  • 验证码识别：使用第三方打码平台或ddddocr模块识别验证码图片。
  • 提交点选：根据识别结果和坐标，构造提交点选验证的请求数据。

  • 登录请求：在点选验证通过后，携带必要的参数和RID发送登录请求。
