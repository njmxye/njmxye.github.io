设计一个有 getMin 功能的栈

题目
实现一个特殊的栈，在基本功能基础上实现返回栈中最小元素的操作。

要求

  1. pop、push、getMin 操作的时间复杂度都是 O(1)。
  2. 设计的栈类型可以使用现成的栈结构。

解答
我们在设计时，使用两个栈：

  • stackData：用来保存当前栈中的元素，功能和正常栈没区别。
  • stackMin：用于保存每一步的最小值。

方案一：压入时省空间，弹出时费时间

  • 压入规则：
    1. stackData 压入 newNum。
    2. 若 stackMin 为空，则 newNum 也压入 stackMin。
    3. 若 stackMin 不为空，比较 newNum 和 stackMin 栈顶：
      • 若 newNum <= stackMin 栈顶，则 newNum 也压入 stackMin。
      • 若 newNum > stackMin 栈顶，stackMin 不压入任何内容。
  • 弹出规则：
    1. stackData 弹出 value。
    2. 若 value == stackMin 栈顶，则 stackMin 弹出。
  • 查询规则：
    • stackMin 始终记录着 stackData 中的最小值，直接返回 stackMin 栈顶。

方案二：压入时费空间，弹出时省时间

  • 压入规则：
    1. stackData 压入 newNum。
    2. 若 stackMin 为空，则 newNum 也压入 stackMin。
    3. 若 stackMin 不为空，比较 newNum 和 stackMin 栈顶：
      • 若 newNum < stackMin 栈顶，则 newNum 压入 stackMin。
      • 若 newNum >= stackMin 栈顶，则重复压入 stackMin 的栈顶元素。
  • 弹出规则：
    • stackData 和 stackMin 同步弹出，总是对应关系。
  • 查询规则：
    • 同方案一，直接返回 stackMin 栈顶。

    • 方案二：stackMin 压入时稍费空间（重复压入），弹出时稍省时间（直接弹出）。

由两个栈组成的队列

题目
编写一个类，用两个栈实现队列，支持队列的基本操作（add、poll、peek）。

解答
利用栈的“先进后出”特性，使用两个栈来实现队列的“先进先出”：

  • stackPush：负责压入数据。
  • stackPop：负责弹出数据。

核心注意事项
数据从 stackPush 压入 stackPop 时，必须严格遵守以下两点，否则顺序会乱：

  1. 必须一次性压入：stackPush 要倒数据，必须把所有数据一次性倒入 stackPop。
  2. stackPop 为空：只有当 stackPop 为空时，才能往里倒数据。

如何仅用递归函数和栈操作逆序一个栈

题目
实现一个栈的逆序，将栈顶到栈底元素 (e.g., 1, 2, 3) 反转为 (3, 2, 1)。

要求

  1. 只能使用递归函数和栈操作。
  2. 不能使用其他数据结构（如列表、数组等）。

解答
设计两个递归函数来解决：

  1. getAndRemoveLastElement：移除并返回当前栈底元素。
    • 利用递归一直向下直至栈底，取出元素后，再将上层元素重新压回。
  2. reverse：逆序整个栈。
    • 利用 getAndRemoveLastElement 拿到栈底元素 i。
    • 递归调用 reverse 处理剩余栈。
    • 最后将 i 压入栈顶。

用一个栈实现另一个栈的排序

题目
将一个栈中的元素从顶到底按“从大到小”的顺序排序（即栈顶最大）。

要求

  1. 只允许申请一个辅助栈。
  2. 可以申请新的变量，但不能使用其他数据结构（如数组）。

解答
使用一个辅助栈 help。

  1. 从原栈 stack 中弹出一个元素 cur。
  2. 如果 cur 小于或等于 help 的栈顶元素，直接压入 help。
  3. 如果 cur 大于 help 的栈顶元素，则将 help 的元素逐一弹出并压回 stack，直到 cur 小于或等于 help 栈顶（或 help 为空），再将 cur 压入 help。
  4. 重复上述过程直到 stack 为空。
  5. 最后将 help 中的所有元素压回 stack，此时 stack 即为从大到小排序。

用栈来求解汉诺塔问题

题目
汉诺塔问题（修改版）：限制不能从最左侧(Left)直接移动到最右侧(Right)，也不能反之，必须经过中间(Mid)。求N层汉诺塔从Left移动到Right的最优移动过程和总步数。

要求

  1. 方法一：递归。
  2. 方法二：非递归，用栈模拟。

方法一：递归

  • 核心逻辑：
    • 由于必须经过中间，移动动作减少为4种：L->M, M->L, M->R, R->M。
    • 相邻移动（如 L->M）：
      1. 1~N-1 层 L -> R (递归)
      2. N 层 L -> M
      3. 1~N-1 层 R -> M (递归)
    • 跨越移动（如 L -> R）：
      1. 1~N-1 层 L -> R (递归)
      2. N 层 L -> M
      3. 1~N-1 层 R -> L (递归)
      4. N 层 M -> R
      5. 1~N-1 层 L -> R (递归)

代码实现 (Recursive)

方法二：非递归 (栈模拟)

  • 核心原则：
    1. 小压大：移动必须满足小盘压大盘。
    2. 相邻不可逆：如果上一步是 L->M，这一步绝不能是 M->L（反之亦然）。
  • 结论：根据这两个原则，每一步实际上只有唯一的合法移动（除了第一步，第一步显然是最小盘 L->M）。
  • 实现：使用三个栈 (lS, mS, rS) 模拟三个塔，记录上一步动作，循环执行直到完成。

生成窗口最大值数组

题目
有一个整型数组 arr 和一个大小为 w 的窗口从数组的最左边滑到最右边，窗口每次向右边滑一个位置。
如果数组长度为 n，窗口大小为 w，则一共产生 n-w+1 个窗口的最大值。
请实现一个函数。

  • 输入：整型数组 arr，窗口大小为 w。
  • 输出：一个长度为 n-w+1 的数组 res，res[i] 表示每一种窗口状态下的最大值。

举例
数组为 [4,3,5,4,3,3,6,7]，窗口大小为 3 时：


结果应返回 {5,5,5,4,6,7}。

解答
假设数组长度为 N，窗口大小为 w。

  • 暴力解法：遍历每个窗口，遍历窗口内元素求最大值，时间复杂度 O(N*w)。
  • 最优解法：使用双端队列 (Deque)，时间复杂度 O(N)。

双端队列 (qmax) 更新规则：
qmax 存放数组 arr 中的下标。

  1. 放入规则：
    • 当遍历到 arr[i] 时，如果 qmax 为空，直接将 i 放入队尾。
    • 如果 qmax 不为空，取出队尾下标 j。
      • 如果 arr[j] > arr[i]，直接将 i 放入队尾。
      • 如果 arr[j] <= arr[i]，将 j 从队尾弹出，重复此过程，直到 qmax 为空或队尾对应值大于 arr[i]，然后将 i 放入队尾。
    • (保证队列中下标对应的数值是从大到小的)
  2. 弹出规则：
    • 如果 qmax 队头的下标等于 i - w，说明队头下标已过期（滑出窗口），将其弹出。
  3. 记录结果：
    • 当 i >= w - 1 时，窗口形成，arr[qmax.peekFirst()] 即为当前窗口的最大值。

题目
给定一个不含有重复值的数组 arr，找到每一个 i 位置左边和右边离 i 位置最近且值比 arr[i] 小的位置。返回所有位置相应的信息。

给定一个可能含有重复值的数组 arr，找到每一个 i 位置左边和右边离 i 位置最近且值比 arr[i] 小的位置。

举例
arr = {3, 4, 1, 5, 6, 2, 7}
返回结果如下（res[i][0] 为左边最近较小值下标，res[i][1] 为右边最近较小值下标，-1 表示不存在）：

解答

  • 核心思想：使用单调栈，栈中存放数组下标，代表的值从栈底到栈顶严格递增。
  • 逻辑：
    1. 遍历数组，当 arr[i] > 栈顶代表的值时，直接入栈（保持递增）。
    2. 当 arr[i] < 栈顶代表的值时，说明栈顶元素的右边最近较小值出现了（即 i）。
      • 弹出栈顶 popIndex。
      • 右边最近较小值：当前遍历到的 i。
      • 左边最近较小值：弹出后的新栈顶（如果栈为空则为 -1）。
      • 重复直到栈顶 <= arr[i] 或栈为空，将 i 入栈。
    3. 遍历完成后，清算栈中剩余元素：
      • 右边最近较小值：-1（不存在）。
      • 左边最近较小值：弹出后的新栈顶（如果栈为空则为 -1）。

进阶（有重复值）

  • 当 arr[i] == 栈顶代表的值时，将下标合并在一起（栈存储 List<Integer> 或处理相等情况不弹出，等后面更小值出现时一起结算）。
  • 通常做法是：栈中存储链表/列表 List<Integer>，代表值相同的一组下标。
    • 弹出时，列表中所有下标的右边最近较小值都是 i。
    • 列表中所有下标的左边最近较小值都是栈中下一个列表的最后一个下标（即最晚加入的那个）。

求最大子矩阵的大小

题目
给定一个整型矩阵 map，其中的值只有 0 和 1 两种，求其中全是 1 的所有矩形区域中，最大的矩形区域为 1 的数量。

举例


结果返回 3。

结果返回 6。

解答

  • 时间复杂度：O(N * M)，其中 N 为行数，M 为列数。
  • 核心思路：将问题转化为求直方图中的最大矩形面积。
    1. 逐行切割：以每一行作为底，统计每个位置往上连续 1 的数量，形成一个高度数组 height。
      • 如果 map[i][j] == 0，则 height[j] = 0。
      • 如果 map[i][j] == 1，则 height[j] = 上一行height[j] + 1。
    2. 计算最大面积：对于每一行的 height 数组（看作直方图），利用单调栈求解其中的最大矩形面积。
      • 对于直方图中的每一根柱子 x，找到其左边最近比它小的位置 k，和右边最近比它小的位置 j。
      • 该柱子能扩展的最大面积 = (j - k - 1) * height[x]。
    3. 全局最大：每一行计算出的最大值中取最大。

最大值减去最小值小于或等于 num 的子数组数量

题目
给定数组 arr 和整数 num，共返回有多少个子数组满足如下情况：
max(arr[i..j]) - min(arr[i..j]) <= num
max(arr[i..j]) 表示子数组 arr[i..j] 中的最大值，min(arr[i..j]) 表示子数组 arr[i..j] 中的最小值。

要求
如果数组长度为 N，请实现时间复杂度为 O(N) 的解法。

解答

  • 核心结论：
    1. 如果子数组 arr[i..j] 满足条件，则 arr[i..j] 中的每一个子数组也都满足条件（因为范围缩小，最大值不会变大，最小值不会变小，差值只会变小或不变）。
    2. 如果子数组 arr[i..j] 不满足条件，则所有包含 arr[i..j] 的子数组也都不满足条件（因为范围扩大，最大值可能变大，最小值可能变小，差值只会变大或不变）。
  • 算法流程 (双端队列 + 滑动窗口)：
    1. 使用两个双端队列 qmax 和 qmin 分别维护窗口内的最大值和最小值的更新（存储下标）。
    2. 使用两个变量 i (左边界) 和 j (右边界)，初始均为 0。
    3. 固定 i，让 j 不断向右扩，更新 qmax 和 qmin。
    4. 当 arr[i..j] 刚好不满足条件（即 max - min > num）或者 j 越界时，停止扩大。此时，以 i 开头的满足条件的子数组数量为 j - i 个（即 arr[i..i], arr[i..i+1] ... arr[i..j-1]）。
    5. 累加数量 res += j - i。
    6. i 向右移动一位 (i++)，并检查 qmax 和 qmin 的队头是否过期（等于 i-1 则弹出），然后继续尝试向右扩展 j。
    7. 直到 i 遍历完整个数组。
  • 时间复杂度：i 和 j 都只增加不减少，每个元素最多进出队列一次，复杂度为 O(N)。

可见的山峰对数量

题目
一个不含有负数的数组可以代表一圈环形山，每个位置的值代表山的高度。比如 {3,1,2,4,5} 代表一个环形结构。
山峰 A 和 山峰 B 能够相互看见的条件为：

  1. 如果 A 和 B 是同一座山，认为不能相互看见。
  2. 如果 A 和 B 是不同的山，并且在环中相邻，认为可以相互看见。
  3. 如果 A 和 B 是不同的山，并且在环中不相邻，假设两座山高度的最小值为 min。如果 A 通过 next 方向到 B 的途中没有高度比 min 大的山峰，或者 A 通过 last 方向到 B 的途中没有高度比 min 大的山峰，认为 A 和 B 可以相互看见。

问题

  1. 原问题：如果 arr 中没有重复值，返回有多少对山峰能够相互看见。
  2. 进阶问题：如果 arr 中含有重复值，返回有多少对山峰能够相互看见。

要求
如果 arr 长度为 N，没有重复值的情况下时间复杂度达到 O(1)，可能有重复值的情况下时间复杂度请达到 O(N)。

解答

1. 原问题 (无重复值)

  • 结论：2 * N - 3 对。
  • 推导：
    • 环形结构中最高的山峰（记为 Max）和次高的山峰（记为 SecondMax）一定能互相看见（1对）。
    • 除了 Max 和 SecondMax 之外，剩下的 N - 2 座山峰，每一座山峰 X 在顺时针和逆时针方向上，一定会遇到第一座比它高的山峰。因为没有重复值，所以这两个方向找到的比它高的山峰必然不同。
    • 所以这 N - 2 座山峰，每座都能贡献 2 对可见山峰对。
    • 总数 = 1 + (N - 2) * 2 = 2 * N - 3。
    • (前提 N >= 2，如果 N < 2 答案为 0)。

2. 进阶问题 (有重复值)

  • 核心思路：使用单调栈。
    1. 准备工作：找到数组中最大值的位置，将其作为遍历的起始点，保证栈底永远是最大值，防止逻辑断裂。
    2. 单调栈策略：栈中存放 Record 对象（包含高度 value 和出现次数 times），保持栈底到栈顶高度严格递减。
    3. 遍历流程：
      • 当前高度 cur > 栈顶高度 top：
        • 弹出栈顶 Record(k, times)。
        • 结算：该记录产生的可见对数为 C(2, times) + times * 2。
          • C(2, times)：相同高度内部互看（如果 times=1 则为0）。
          • times * 2：对外看，顺时针遇到 cur，逆时针遇到栈中下一个元素（因为是从最大值开始遍历，栈底总是最大值，所以一定存在下一个元素）。
      • 当前高度 cur == 栈顶高度 top：栈顶 times++。
      • 当前高度 cur < 栈顶高度 top：压入新记录。
    4. 清算阶段（遍历结束后栈不为空）：
      • 第 1 阶段：弹出记录不是倒数第 1 条，也不是倒数第 2 条。
        • 产生对数：C(2, times) + times * 2。
      • 第 2 阶段：弹出记录是倒数第 2 条。
        • 如果倒数第 1 条（最大值）的 times == 1，产生对数 C(2, times) + times * 1（因为顺时针和逆时针看到的是同一个最大值）。
        • 如果倒数第 1 条（最大值）的 times > 1，产生对数 C(2, times) + times * 2。
      • 第 3 阶段：弹出记录是倒数第 1 条（最大值）。
        • 产生对数：C(2, times)（内部互看，对外不再产生对数）。
